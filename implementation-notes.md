# 舞鶴 PWA 改版實作紀錄 (Implementation Notes)

本紀錄詳細記載本次 PPT 八項修改改版中，技術架構與需求對齊時的待釐清事項、決定、偏離與取捨。

---

## 1. 待釐清 (Clarifications)
*   **卡片「橫式」定義釐清**：
    *   **原先疑問**：PPT 提到「橫式 (landscape)」，但提供的背景圖 `crane_front_v2.jpg` and `crane_back_v2.png` 均為直式。若強行將卡片外觀修改為寬扁的橫向比例，底圖文字將會被截斷一半以上。
    *   **釐清結果**：經與您確認，此處的「橫式」指的是「**卡片內部籤詩文字的書寫方向**」（由直書改為橫書，即左至右水平排版），而「**卡片外觀本體維持原本的直式比例**」。這讓我們能夠完整且無裁切地呈現 `crane_front_v2.jpg` 的木雕文字和 `crane_back_v2.png` 的精美深色木框。

## 2. 決定 (Decisions)
*   **導覽列對稱與手機防擠壓排版**：
    *   **決定**：因為新增「遊戲 🕊」與「好物 🛍」兩個分頁，為避免在 375px 小螢幕（如 iPhone SE）下 5 個按鈕重疊擠壓，我們將其重新排版，改為左右對稱的 Nav Group 佈局：
        *   左邊：`[ 收藏 ♥ ] [ 地圖 🗺 ]`
        *   中間：`[ 今日舞鶴 ]` (Logo 居中且不可縮減)
        *   右邊：`[ 遊戲 🕊 ] [ 好物 🛍 ]`
    *   並在媒體查詢中，當寬度小於 380px 時，自動微調字體大小為 11.5px、按鈕 padding 縮小，並設定 `white-space: nowrap;` 以確保不折行且保持精美對稱的版面。
*   **引言全景圖滿版展示**：
    *   **決定**：使用 HTML `<img>` 代替 CSS 背景圖。設定 `width: 100%; height: auto; display: block; object-fit: cover;`，利用其自身 1604x670 的全景比例滿版呈現，不需要寫死高度，防止任何圖片文字遭截。

## 3. 偏離 (Deviations)
*   **原住民族名稱取代無符合結果**：
    *   **偏離**：第六項要求「全站搜尋所有提到『原住民族』並取代為『撒奇萊雅族』」。
    *   **原因**：經過 Grep 搜尋全站原始碼與 JSON 資料，發現目前的程式碼中**完全沒有**出現「原住民族」或「原住民」字串。而在地圖第四篇中，已經精準寫明「撒奇萊雅族」，其餘地圖篇章則是具體阿美族語。因此本項偏離原訂的「取代」行為，改為維持現狀。

## 4. 取捨 (Trade-offs)
*   **刪除重複籤詩區塊與收藏狀態保留**：
    *   **取捨**：原本翻牌後，卡片內顯示一次籤詩，卡片外解籤區也顯示一次籤詩。在移除外部重複籤詩區後，我們將卡片背面的文字內容與解籤內容整合到同一個 3D 卡片正面中。
    *   **技術考量**：我們捨棄了原先在解籤區呈現的備份 DOM 節點，改為動態渲染卡片內的各個子節點。這樣一來，外層的幸運指數、小行動、收藏與分享邏輯依然依據 localStorage 的 `lastFortuneId` 運作，既保持了程式碼的簡潔與無重複，又完整保留了收藏清單隨存隨取的功能，完全不影響舊用戶的收藏資料。
