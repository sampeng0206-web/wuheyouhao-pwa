# 舞鶴 PWA 第四輪改版實作紀錄 (Implementation Notes)

本紀錄詳細記載本次第四輪改版（依用戶反饋 12 項修改）中，技術架構與需求對齊時的待釐清事項、決定、偏離與取捨。

---

## 1. 待釐清 (Clarifications)
*   **掃叭石柱說明文字之阿美語陳述**：
    *   **釐清點**：第十一項要求全站將「阿美族」改為「撒奇萊雅族」，但第四項掃叭石柱說明中提到 `Satokoay考古遺址為阿美語（石柱所在之地）`。
    *   **釐清結果**：經與您確認，此句為學術/地名歷史語言來源考證陳述，不受第十一項全站替換的限制，因此在第四項中精準保留「阿美語」，避免歷史事實被誤改。
*   **舞鶴木牌小卡片雙圖並列排版限制**：
    *   **釐清點**：手機版好物小卡片（寬度約 150px~170px）若採左右 50% 並列顯示雙圖，會導致木牌的雕刻細節與文字完全看不清楚，影響視覺觀感。
    *   **釐清結果**：經綜合考量，我們採用了「垂直堆疊（Stack）雙圖」的設計，在專門的 `.tag-img-wrapper` 內同時直向展示正面與反面兩張大圖，不僅細節保留完好，又避免了複雜的 JavaScript 點擊放大彈窗邏輯。

---

## 2. 決定 (Decisions)
*   **導覽列文字字級階層（長輩友善）**：
    *   **決定**：為了優化長輩閱讀體驗，「收藏、地圖、遊戲、好物」四個按鈕的字體大小從 `13px` 提升到 `16px` 並設定為 `font-weight: 700`；同時為了維持視覺上的主從關係，「今日舞鶴」Logo 字體大小同步從 `16px` 提升至 `22px`，在手機版也透過 media query 同步微調為 `14px` 與 `18px`，在增強清晰度的同時，兼顧排版的對稱性與精美感。
*   **翻牌畫面黑字與去除陰影**：
    *   **決定**：為了解決長輩反映的卡片背面文字模糊問題，將 `.card-fortune-title`、`.card-fortune-quote`、`.card-fortune-interpretation`、`.card-fortune-ending` 等卡片內所有橫書文字的 `color` 一律改為純黑色 (`#000000`)，並將 `text-shadow` 設為 `none`。這使文字與底圖 `crane_back_v2.png` 產生最高對比度，底圖浮水印（木雕花紋）被自然呈現在文字之下。
*   **縮短「心靈鶴湯」與解籤的間距**：
    *   **決定**：將卡片背面的 HTML 結構重組，將標題與籤詩歸入 `.card-top-group`，將分隔線、解籤內文、結尾詞歸入 `.card-bottom-group`。搭配 CSS 設定 `.card-bottom-group` 的 `margin-top: auto` 以及元素間極小 gap，使解籤內容自然下移並緊密貼合，拿掉了原先 `space-between` 所造成的多餘空白。

## 3. 偏離 (Deviations)
*   **全站「阿美族」替換之範圍限定**：
    *   **偏離**：第十一項指示將全站指涉「阿美族」族群認定錯誤之處改為「撒奇萊雅族」。
    *   **原因**：除了第四項掃叭石柱的「阿美語」考證維持原樣外，原本第二篇的 Kalala 篇包含「阿美族語中藤編籃子的意思」、「阿美族先民用月桃葉」等字樣，但因本輪修改已將 Kalala 篇「整篇替換」為「北回歸線標誌公園」，因此這些段落直接被整段移除，不需進行逐字替換；實際修改範圍為 JS 寶物列表中的「阿美族獵人的勇氣石」改為「撒奇萊雅族獵人的勇氣石」。

---

## 4. 取捨 (Trade-offs)
*   **九宮格循環收集與勳章累計設計**：
    *   **取捨**：舊版的九宮格是靜態 DOM（固定 9 格），收集到 9 種寶物後，第四天起尋寶若抽到新寶物會直接被 `localStorage` 排除或因 DOM 無法渲染而消失。
    *   **解決方式**：我們捨棄了硬編碼 9 格的做法，改為用 JavaScript 動態生成網格 DOM：
        1. 網格最少維持 9 格（若空缺以 `?` 佔位），當前收集長度若大於 9 則自適應擴充，徹底杜減寶物溢出消失的錯誤。
        2. 當玩家集齊 9 個不同的寶物時，彈出勳章視窗並提供「收下勳章並開啟新一輪」按鈕。點擊後會把 active 寶物備份寫入歷史圖鑑 `craneAllTimeCollection`，隨後清空當前收集包並使累計勳章數（🏅 x N）+1，重新開啟新一輪。這在完美保留歷史足跡的同時，又提供了一個能讓玩家重複遊玩且不漏掉新寶物的永續遊戲循環。
*   **Service Worker 與 App 快閃舊 Logo 版本版控**：
    *   **取捨**：為解決 Android 手機點擊啟動 PWA 時快閃舊 Logo 的快取問題，我們採取了雙重版本控制措施：
        1. 在 `manifest.json` 與 `index.html` 中的 Logo icon 路徑加上 query parameters (如 `icon-192.png?v=4` 及 `manifest.json?v=4`)，強制瀏覽器在偵測到 manifest 或 icon 版本更新時重建 WebAPK。
        2. 在 `sw.js` 的 activate 事件中加入 `self.clients.claim()`，並將快取版本升級為 `wuheyouhao-v15`，刪除廢棄的 7 張舊圖，加入 14 張新圖，確保快取資源第一時間接管，排除舊版本渲染。
